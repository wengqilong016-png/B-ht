# 管理端司机管理操作手册

> BHT 管理后台 — 车队司机增删改查、薪资核算、搜索筛选分页操作指南

---

## 1. 添加司机

### 目的

在系统中注册新司机，创建登录账号、个人信息和薪资方案。

### 步骤

1. 进入「车队管理」页面，点击右上角 **Add New Driver** 按钮。
2. 在弹出的表单中填写以下信息：

   **必填项：**

   | 字段 | 说明 |
   |------|------|
   | 姓名 (Name) | 司机真实姓名，必填。输入后自动生成 Driver ID 和登录邮箱。 |
   | 初始密码 (Password) | 司机首次登录用的密码，必填，至少 8 位。 |

   **自动生成：**

   | 字段 | 规则 |
   |------|------|
   | 登录邮箱 | 由姓名自动生成：`姓名@bht.com`。例如输入 `张三` → `zhang.san@bht.com`。司机首次登录后可自行绑定真实邮箱。 |
   | 司机 ID (Driver ID) | 由姓名自动生成（大写+下划线），也可手动修改。留空则自动生成随机 UUID。 |

   **选填项：**

   | 字段 | 说明 | 默认值 |
   |------|------|--------|
   | 电话 (Phone) | 司机联系电话 | 空 |
   | 车辆型号 (Vehicle Model) | 如 Bajaj、TVS | 空 |
   | 车牌号 (License Plate) | 如 T 000 XXX | 空 |
   | 流动硬币 (Floating Coins) | 司机随身携带的硬币数（TZS），每日结算后自动更新 | 10,000 |
   | 月基本工资 (Base Salary) | TZS | 300,000 |
   | 提成比例 (Commission Rate) | 百分比 | 5% |
   | 初始欠款 (Initial Debt) | 创建时的欠款金额 | 0 |

3. 点击 **Save Driver Profile** 提交。
4. 系统在后台依次执行：
   - 通过 Edge Function 创建 Supabase Auth 用户 + drivers 表记录 + profiles 表记录
   - 回填业务字段（电话、车辆、薪资方案等）
5. 成功后显示成功提示，新司机卡片出现在列表中。

### 注意事项

- 密码至少 8 位，否则无法提交。
- 姓名不能为空。
- 如果业务字段（电话、薪资等）回填失败，账号仍会创建成功，但需管理员手动编辑补填——系统会弹出警告提示。
- 如果邮箱已被注册（`EMAIL_CONFLICT`）或司机 ID 已存在（`DRIVER_ID_CONFLICT`），会显示明确错误提示。
- Edge Function 调用有 30 秒超时，超时后请检查网络重试。

## 2. 编辑司机

### 目的

修改司机个人信息、薪资方案、账号状态，以及分配/解绑机器点位。

### 步骤

#### 2.1 修改信息

1. 点击司机卡片上的 **编辑** 按钮（或分析视图中对应行的编辑按钮）。
2. 在弹出表单中修改需要变更的字段：
   - 姓名、电话、车辆型号、车牌号
   - 流动硬币、月基本工资、提成比例
   - 当前欠款（仅在编辑模式下显示，可直接修改）
3. 点击 **Save Driver Profile** 保存。
4. 系统通过 upsert 全量更新 drivers 表，并乐观更新前端缓存。

#### 2.2 切换账号状态 (Active / Inactive)

有两种方式：

- **方式一（表单内）**：编辑司机时，在「账号状态」区域点击开关按钮，Active ↔ Inactive 即时切换，随表单一起保存。
- **方式二（卡片快捷操作）**：在司机卡片上直接点击状态切换按钮，会弹出确认对话框，确认后立即生效。

#### 2.3 分配机器点位

仅在编辑已有司机时可用：

1. 编辑司机，滚动到「分配机器点位 Assign Locations」区域。
2. 勾选/取消勾选需要分配或解绑的机器。每台机器旁显示状态灯：
   - 绿色 = 正常运营 (active)
   - 黄色 = 维护中 (maintenance)
   - 红色 = 故障 (broken)
3. 保存后，系统并发更新司机信息和机器分配关系。

### 注意事项

- 编辑模式下**不显示密码字段**——如需重置密码请通过其他途径。
- 更改机器分配时，`updateDrivers` 和 `updateLocations` 并发执行，非事务性。如果其中一个失败，另一个可能已生效。
- 状态切换仅在 active 和 inactive 之间切换。

## 3. 删除司机

### 目的

从系统中永久移除司机及其登录凭据。

### 步骤

1. 点击司机卡片上的 **删除** 按钮。
2. 系统弹出确认对话框：

   > 此操作将永久删除登录凭据及所有关联数据，不可撤销。
   > Delete this driver? This will permanently remove their login credentials. This cannot be undone.

3. 点击「确认删除」执行删除。
4. 系统调用 Edge Function `delete-driver`，依次删除：
   - Supabase Auth 用户（级联删除 profiles）
   - drivers 表记录
5. 成功后司机从列表中消失。

### 注意事项

- ⚠️ **必须在线**才能删除。离线时删除按钮不响应，并提示「离线无法删除」。
- ⚠️ 删除司机后，以下数据受影响：

| 关联数据 | 影响 |
|----------|------|
| 机器分配 (locations.assignedDriverId) | 自动解除绑定 (SET NULL) |
| 历史交易 (transactions.driverId) | 保留交易但失去司机关联 (SET NULL) |
| 日结算 (daily_settlements.driverId) | 失去关联 (SET NULL) |
| **月工资单 (monthly_payrolls)** | **⚠️ 物理删除 (CASCADE)**，工资数据永久丢失 |
| 换点申请、健康报告、告警 | 失去关联 (SET NULL) |

- 删除前请确认该司机的工资单已导出/归档，以免数据丢失。
- 删除操作不可撤销。

## 4. 薪资中心

### 目的

快速核算某位司机当月的应发工资，包含基本工资、提成、扣款项。

### 步骤

1. 在司机卡片上点击 **薪资** 按钮（或分析视图中点击对应按钮）。
2. 弹出「月度工资单」弹窗，显示：

| 项目 | 计算方式 |
|------|----------|
| 当月营收 | 当月所有已确认日结算（confirmed）的 totalRevenue 之和 |
| 收款次数 | 当月已付 (paid) 的 collection 类型交易数 |
| 基本工资 | 司机档案中配置的 baseSalary |
| 提成 | 当月营收 × commissionRate（向下取整） |
| 私人借款扣减 | 当月 expenseType=private 的 expenses 之和 |
| 短缺扣减 | 当月结算中 shortage 为负的绝对值之和 |
| **净工资** | 基本工资 + 提成 − 借款 − 短缺 |

3. 点击「确认关闭」退出。

### 注意事项

- 薪资核算为**前端即时计算**，供参考使用，非正式工资单。
- 计算范围为**当前自然月**，数据来源为已确认的日结算和已付的收款交易。
- 提成比例来自司机档案的 `commissionRate`（编辑司机可修改）。
- 计算结果不以任何形式持久化到数据库——仅供管理员现场查看。

## 5. 搜索 / 排序 / 分页

### 目的

在海量司机中快速定位目标，按关键指标排序和翻页浏览。

### 搜索

- 在顶部搜索框中输入关键词，**实时过滤**司机列表。
- 搜索范围：**姓名**、**司机 ID (username)**、**电话号码**。
- 搜索不区分大小写。
- 清空搜索框恢复完整列表。

### 排序

- 点击搜索框右侧的排序下拉菜单，选择排序字段：
  - **总营收 (Revenue)** — 按司机累计营收排序（默认）
  - **欠款 (Debt)** — 按剩余欠款金额排序
  - **姓名 (Name)** — 按字母/拼音排序
  - **状态 (Status)** — 按 active/inactive 分组排序
- 点击箭头按钮切换 **升序 ↑ / 降序 ↓**。
- 切换排序字段时自动重置为降序。

### 分页

- 网格视图每页显示 **9** 张司机卡片，分析视图每页显示 **12** 行。
- 当司机数量超过一页时，底部出现翻页控件：上一页 / 页码 / 下一页。
- 搜索或排序变更时自动回到第 1 页。

### 视图切换

- 右上角切换按钮：
  - **卡片视图 (Grid)** — 卡片式布局，适合浏览司机概况
  - **分析视图 (Analytics)** — 表格式布局，展示车队统计和详细数据列

### 注意事项

- 搜索、排序、分页均为**前端计算**，无需请求服务器，响应即时。
- 切换视图模式时自动回到第 1 页。
- 当前页码在搜索/排序变更后重置为 1。
