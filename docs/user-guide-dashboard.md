## 1. Dashboard 仪表盘

Dashboard 是管理员登录后的主工作台，顶部有 7 个标签页切换：
**总览 | 网点 | 结算审批 | 车队 | 轨迹追踪 | AI LOGS**

（司机端仅显示「结算」标签页）

### 1.1 总营收卡片

进入 Dashboard 默认显示 **总览** 标签页，左侧最醒目的是 **今日营收卡片**：

- 显示当天所有收款的总金额（TZS）
- 点击卡片可展开按司机分列的营收明细：
  - 每位司机今日收款笔数、总营收、分润金额、实收金额
  - 列出每笔交易的具体机器和金额

### 1.2 异常点位与高风险资产

营收卡片右侧并列两个监控卡片：

| 卡片 | 含义 | 颜色 |
|------|------|------|
| **异常点位** | 状态非 active、被锁、分数≥9000、或长时间无交易的机器 | 灰色 |
| **高风险资产** | 收入方差异常或连续短缺的司机 | 琥珀色 |

点击数字可直接跳转到对应标签页处理。

### 1.3 今日待办中心

总览页中部是 **今日行动中心**，按优先级自动排序列出待处理事项：

| 待办项 | 说明 | 跳转 |
|--------|------|------|
| 审批中心 | 待审核的日结、费用、异常交易、重置请求、提现请求 | 结算审批 |
| 异常点位 | 异常、锁定、9999风险和静默机器 | 网点 |
| 高风险司机 | 收入异常波动的司机 | 车队 |
| 工资单 | 待生成、待支付或待取消的工资单 | 车队 |
| 同步队列 | 待同步记录和轨迹审计异常 | 轨迹追踪 |

- 最紧急的事项以大号深色卡片展示
- 其余待办以小卡片排列在下方
- 全部清零时显示「无紧急工作」提示

### 1.4 AI 智能分析助手

总览页底部有可折叠的 **AI 智能分析** 面板：

- 点击「AI 智能分析」标题栏展开
- 系统自动分析交易数据，生成异常模式、趋势变化等洞察
- 另外 Dashboard 右下角还有 **Bahati AI 助手** 浮动聊天窗口：
  - 支持中文和英文
  - 预设快捷提问：今日概况、查司机、异常检测、营收分析
  - 输入自定义问题获取实时分析
  - 对话历史保留在当前会话中

### 1.5 网点概览（SitesTab）

点击「网点」标签页查看所有机器：

- 搜索框可按机器名称/编号筛选
- 每台机器显示：状态指示灯（绿=active，黄=maintenance，红=broken）、名称、机器编号、最新分数、分润比例
- 管理员可编辑机器信息（分润比例、状态等）
- 删除机器前会进行阻塞检查（未清债务、未付分红、待审批交易等），有强制清除阻塞功能

### 1.6 结算审批（SettlementTab）

点击「结算审批」标签页：

- **管理员视角**：展示所有待审批任务列表
  - 日结对账审核（确认/拒绝）
  - 费用报销审批
  - 异常交易审核
  - 重置请求审批
  - 提现请求审批
- **司机视角**：展示自己的日结提交入口和状态
- 审批操作会级联更新：结算状态 → 关联交易的支付状态 → 司机浮动币

### 1.7 车队管理（Team 标签页）

点击「车队」标签页进入司机管理，详见 `docs/admin-crud-trace.md` 中的 Drivers CRUD 章节。

## 2. 地图与轨迹追踪

点击「轨迹追踪」标签页进入司机位置追踪。

### 2.1 追踪概览卡片

页面顶部显示 4 个状态卡片：

| 卡片 | 含义 |
|------|------|
| **活跃司机** | 当前在线（刚活跃）的司机数 |
| **GPS 过期** | GPS 数据超过阈值未更新的司机数 |
| **今日收银** | 今天全部收款笔数 |
| **关注站点** | 需要注意的异常机器数 |

### 2.2 司机状态筛选

支持 4 种筛选模式：

- **全部** — 不筛选
- **需关注** — 有关注站点或 GPS 过期的司机
- **活跃** — 当前在线的司机
- **失联** — 长时间未活跃的司机

### 2.3 司机追踪卡片

每位司机显示为一张可展开卡片：

- **收起状态**：头像、姓名、管辖机器数、活跃状态标签、今日收款数/金额
- **展开状态**：
  - 四格数据：今日收款笔数、今日现金收入、GPS 时间、关注站点数
  - 管辖机器列表：每台显示状态灯、名称、机器编号、最新分数、分润比例
  - 机器状态标签：maintenance、resetLocked、≥9999 风险、所属区域
  - 点击铅笔图标可快速编辑机器的分润比例和状态
  - 如果司机有 GPS 坐标，显示精度坐标并可一键打开 Google Maps 导航

### 2.4 司机活跃状态判断

| 显示 | 含义 |
|------|------|
| 在线 | 最近几分钟内活跃 |
| 离开 | 一段时间未活跃但未离线 |
| 离线 | 长时间未活跃 |

GPS 过期时显示琥珀色「GPS 过期」标签。

### 2.5 实时地图

页面底部有可展开的 **实时地图** 面板：

- 使用 Leaflet 交互式地图
- 显示司机当前位置标记
- 显示机器位置标记（如果有 GPS 坐标）
- 显示当日交易位置标记
- 支持缩放、拖拽等标准地图操作

### 2.6 GPS 数据说明

BHT 系统的 GPS 并非持续追踪，而是**按需采集**：

- 司机每次提交收款时获取一次 GPS 坐标
- GPS 来源优先级：浏览器实时定位 > 照片 EXIF 提取 > 上下文推测
- 无 GPS 时使用 {0, 0} 占位（地图上不显示）
- 司机端的 `driver.currentGps` 字段存储最近一次位置

## 3. 同步状态说明

### 3.1 什么是 SYNCED

BHT 系统使用 `isSynced` 字段标记数据是否已与服务端同步：

| 状态 | 含义 |
|------|------|
| `isSynced: true` | 数据已成功写入 Supabase 数据库，与服务端一致 |
| `isSynced: false` | 数据仅在本地缓存中，尚未持久化到服务端 |

### 3.2 哪些操作会产生未同步数据

- 离线状态下司机提交收款（进入离线队列）
- 网络中断时管理员编辑机器/司机信息
- 线下操作后尚未联网刷新的数据

### 3.3 同步标签页入口

Dashboard 总览页的「今日行动中心」中：

- **同步队列** 项显示当前未同步记录数
- 点击跳转到「轨迹追踪」标签页查看详情

### 3.4 实时同步机制（Realtime）

BHT 使用 Supabase **Broadcast Channels** 实现实时同步，而非传统的轮询：

- 数据库表变更时，触发器自动广播事件
- 管理端订阅 4 个 channel：
  - `db:transactions` — 交易变更
  - `db:drivers` — 司机变更
  - `db:daily_settlements` — 日结变更
  - `db:locations` — 机器变更
- 收到广播后，React Query 缓存自动失效（250ms 防抖），UI 即时刷新

### 3.5 同步故障排查

如果发现同步异常：

1. **检查网络状态**：页面顶部或底部会显示在线/离线状态
2. **检查未同步计数**：总览页行动中心查看 `unsyncedCount`
3. **刷新页面**：重新加载会触发全量 `refetchQueries`
4. **等待自动恢复**：离线队列在网络恢复后自动冲刷
5. **离线队列满**：localStorage 最大 200 条，超出静默丢弃旧事件

## 4. 月度报表

### 4.1 日结对账

在「结算审批」标签页中：

- 司机每天可提交当日日结
- 管理员审核后确认（confirmed）或拒绝（rejected）
- 审核确认后自动级联：关联的 collection 交易 paymentStatus 更新、司机浮动币更新
- 日结包含：总收入、分润金额、贷款、短缺、实付金额

### 4.2 月度工资单（Payroll）

在「车队」标签页中管理：

- **生成工资单**：选择司机和月份，系统自动汇总该月数据
  - 总营收、佣金、贷款、短缺、净应付
- **支付标记**：确认工资已发放，更新状态为 paid
- **取消工资单**：撤销未支付的工资单

### 4.3 如何查看月度经营数据

1. 进入 Dashboard → 总览页 → 点击营收卡片 → 展开查看每位司机当天营收明细
2. 进入「车队」标签页 → 选择司机 → 查看该司机历史交易、日结记录
3. 使用 **Bahati AI 助手** 提问：「分析本周收入趋势，对比上周」
4. 查看 `monthly_payrolls` 表获取汇总数据（需要数据库直连或 Supabase Dashboard）

### 4.4 数据导出建议

当前版本未内置 CSV 导出功能，可通过以下方式获取报表：

- Supabase Dashboard → SQL Editor → 直接查询 `transactions` / `daily_settlements` / `monthly_payrolls` 表
- 使用 AI 助手生成文本摘要后再手动整理

## 5. 操作记录

### 5.1 审计日志体系

BHT 有两套审计日志：

#### finance_audit_log 表

记录所有财务敏感操作，**追加写入，不可修改或删除**：

| 事件类型 | 触发场景 |
|---------|---------|
| `driver_salary_change` | 修改司机基础工资 |
| `driver_commission_change` | 修改司机佣金比例 |
| `driver_debt_edit` | 修改司机债务（initialDebt/remainingDebt） |
| `driver_status_change` | 修改司机状态（active/suspended等） |
| `floating_coins_change` | 修改司机每日浮动币 |
| `commission_rate_change` | 修改机器分润比例 |
| `startup_debt_edit` | 修改机器启动债务 |
| `startup_debt_recovery` | 机器债务回收 |
| `force_clear_blockers` | 管理员强制清除机器删除阻塞 |
| `location_delete` | 删除机器 |

每条记录包含：
- 操作人（actor_id）
- 操作对象（entity_type + entity_id + entity_name）
- 变更前后的值（old_value → new_value）
- 额外上下文（payload JSONB）
- 时间戳（created_at）

#### driver_flow_events 表

记录司机操作流程的完整遥测数据：

- 从选择机器 → 拍照 → 输入金额 → 确认提交的完整步骤链
- 记录 GPS 权限状态、是否有照片、步骤耗时
- 错误分类、在线/离线状态
- **不可修改或删除**，通过 `flow_id` 可重建单次操作的时间线

### 5.2 如何查询审计日志

**通过 Supabase Dashboard**（推荐）：

```sql
-- 按实体查询：某司机的所有变更记录
SELECT * FROM finance_audit_log
WHERE entity_type = 'driver' AND entity_id = 'driver-xxx'
ORDER BY created_at DESC;

-- 按操作人查询：某个管理员的所有操作
SELECT * FROM finance_audit_log
WHERE actor_id = 'admin-xxx'
ORDER BY created_at DESC;

-- 查询司机操作流程：某次收款从开始到结束
SELECT step, event_name, duration_ms, created_at
FROM driver_flow_events
WHERE flow_id = 'xxx'
ORDER BY created_at;

-- 统计提交失败率
SELECT driver_id, COUNT(*)
FROM driver_flow_events
WHERE step = 'complete' AND event_name = 'submit_failed'
GROUP BY driver_id
ORDER BY COUNT(*) DESC;
```

### 5.3 审计日志权限

- **管理员**：可读取全部审计日志（finance_audit_log SELECT 策略仅限 admin）
- **司机**：无法读取审计日志
- **写入**：通过数据库触发器自动写入（driver 字段变更）、或前端 fire-and-forget 写入（机器操作）

### 5.4 注意事项

- 审计日志**不可物理删除**（没有 UPDATE/DELETE 策略）
- `finance_audit_log` 使用 SECURITY DEFINER 触发器，确保即使调用者无直接写权限也能记录
- 离线期间的遥测事件暂存 localStorage（上限 200 条），恢复在线后批量冲刷
