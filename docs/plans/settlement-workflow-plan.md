# BHT 日结流程改进方案

> 2026-05-11 | 秋风

---

## 现状

司机日结页面 (`SETTLEMENT` tab)：
- 显示日期 + 今日收款笔数 + 营收 + 预期现金
- 输入实际收到的现金、硬币
- 录入关店费用（类型 + 金额 + 备注）
- 提交后 → 管理端审批中心 → 确认/拒绝

**问题：**
1. 日结提交按钮只依赖"有没有 collection"，没提示"今天没收款不能日结"
2. 提交后司机端没有状态追踪——不知道审批过没过
3. 管理端审批和工资发放割裂在不同页面
4. 司机不知道日结是干嘛的——提示是英文 Swahili 混排

---

## 改进方案

### 一、司机端 SETTLEMENT 页面改造

**当天没收款时：**
显示引导文字 "今日尚未收款，请前往 HARAKA 或 COLLECT 完成收款后再来日结"
+ 快捷按钮 "去收款 →"

**当天有收款时：**
1. 顶部显示今日汇总卡片：
   - 已收 X 台机器 · 总营收 TZS XXX · 应缴现金 TZS XXX
2. 输入区（已有）：
   - 实际收到现金（纸钞）
   - 实际硬币数
3. 费用录入（已有）：
   - 类型 + 金额 + 备注
4. 差额自动计算：
   - 应缴 vs 实缴 → 短款/盈余 红色/绿色显示
5. **提交按钮始终可用**（只要今天有收款）
6. 提交后 → 页面切换到"等待审批"状态
   - 显示审批状态卡片：🟡 待审批 / 🟢 已确认 / 🔴 已拒绝
   - 显示提交时间、预计处理时间
   - 管理员备注可见

### 二、管理端审批 + 支付一体化

在管理端"车队与薪资"页面，每张司机卡片新增状态条：
- 今日已收 X 笔 · 已日结 ✓ / 待日结 📋
- 点击 → 跳转审批中心查看详情

审批中心保持现有逻辑不变。

### 三、日结完成后的发薪流程

```
日结确认 → 月度自动汇总 → 管理端"车队与薪资"点"薪资中心"
  → 查看工资明细 → 点"标记已支付" → 上传凭证 → 完成
```

## 实施优先级

| 优先级 | 改动 | 影响 |
|--------|------|------|
| P0 | 日结页面显示今日汇总 | 司机知道今天收了多少钱 |
| P0 | 提交后显示审批状态 | 司机知道钱到哪步了 |
| P1 | 没收款时显示引导 | 减少迷惑 |
| P1 | 管理端司机卡片加日结状态 | 一眼看到谁没结 |
| P2 | 差额自动计算 | 少算错 |
